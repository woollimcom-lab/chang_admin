# 프로젝트 전용 Copilot 지침

이 저장소는 커튼/블라인드 설치업체를 위한 현장 서비스 ERP 백엔드(FastAPI + SQLAlchemy) 프로젝트입니다. Copilot 에이전트는 다음 규칙을 준수해야 합니다.

1. **언어 및 커뮤니케이션**
   - 기본 응답은 한국어로 작성합니다. 코드, 명령어, 파일명 등 기술적 용어는 영어로 유지합니다.
   - 출력에서 코드 블록은 항상 ` ``` `로 감싸고 구문 하이라이팅을 적절히 사용합니다.

2. **코드 스타일 및 변경 범위**
   - 기존 코드 스타일(PEP8 준수, 함수 사이에 빈 줄 한 줄, `import` 정렬 등)을 그대로 따릅니다.
   - 전체 파일 재포맷팅은 금지합니다. 변경은 **최소한의 diff** 형태로 제시해야 합니다.
   - 새로 생성하는 파일도 UTF-8 인코딩을 사용합니다.
   - 기존 코드에 한국어 주석이 있으면 새 코드에도 자연스럽게 한국어를 사용합니다.

3. **한국어 UI 텍스트**
   - HTML 템플릿, JavaScript, Python 코드 어디에서든지 한국어 UI 문구를 수정하거나 번역하지 않습니다.
   - 한국어 텍스트는 버그 수정 목적이 아니라면 그대로 둡니다.

4. **프레임워크/아키텍처 관행**
   - FastAPI 라우터(`routers/`), 서비스(`services/`) 및 모델(`models.py`) 구조를 준수합니다.
   - 데이터베이스 연결은 `database.get_db()` 의존성 주입을 사용합니다.
   - 비즈니스 로직은 서비스 모듈에 두고 라우터는 최대한 얇게 유지합니다.
   - `HTTPException`, `JSONResponse` 등의 FastAPI 표준을 활용합니다.
   - 비밀번호 해시는 현재 단순화되어 있으므로 테스트 코드/로직에서 주석 처리된 상태를 그대로 유지합니다.

5. **질문 및 검토**
   - 변경 전에는 영향 범위(scope)와 blast radius를 2~5줄로 설명합니다.
   - 리뷰나 수정 요청이 있으면 문제점, 수정 포인트, 검수 포인트, 리스크 등을 구조화해서 제시합니다.

6. **추가 권장 사항**
   - 새로운 기능을 구현할 때는 먼저 유사한 패턴이 있는지 찾아보고 그에 맞춰 재사용합니다.
   - 불필요한 종속성 추가를 자제하고 `requirements.txt`를 업데이트해야 할 경우 항상 사용자에게 알립니다.

> 위 지침은 지속적으로 발전할 수 있으며, 이후 대화에서 새로운 규칙이나 예외가 도출되면 해당 내용을 이 파일에 반영하세요.
